# Time Sovereignty Competition Demo Script

## One-sentence thesis

Time Sovereignty is a longitudinal AI Chief of Staff that protects a goal when
real life interrupts, instead of merely generating a plan once.

## Demo path

1. Create the Build Week goal, deadline, and motivation.
2. Review the Goal Architect plan and confirm the support agreement.
3. Enter the command center and show the North Star, protected minimum, next
   check-in, and resume point.
4. Open Incoming Check-in. Play the tap-to-play voice message and show the
   quiet-hours boundary.
5. Delay once. Show that it is accepted without judgment.
6. Delay again or choose “Something changed.” Show Commitment Recovery asking
   whether timing, task size, method, or direction is wrong.
7. Confirm a smaller action and show the updated resume point and memory.
8. Share progress as text, photo, or voice. Show specific feedback and the
   stored progress memory.
9. Rate whether the intervention helped.
10. Run the 30-day accelerated simulation. Open Journey to show planning,
    interruption, adaptation, evidence, calibration, and continuation.
11. Open Developer mode. Show need-based Agent traces, schema names, provider
    labels, returned models, and the absence of raw prompts or private
    reasoning.
12. Show the deployed health endpoint, one live GPT-5.6 Cloud Task receipt,
    four persisted `openai` traces, and the duplicate request proof.

## Honest disclosure

- Accelerated time is clearly labeled.
- Local development uses deterministic mock contracts.
- The final cloud acceptance uses GPT-5.6 through the same schemas.
- Browser TTS and speech recognition depend on browser capabilities and user
  permission; text fallback remains available.
- Text, photo, and voice are the competition progress formats. Video and
  generic files are intentionally deferred under Decision 0008.
