# Time Sovereignty — final under-three-minute video plan

Target: **2:50–2:55**. Hard stop: **2:58**. Use English narration or an
AI-assisted English voice, corrected English captions, and no copyrighted
music. The public YouTube upload must remain under three minutes.

## Story rule

Sell one transformation, not a feature list:

> A person who bought art supplies but kept postponing drawing completed a real
> cup sketch after one protected 20-minute commitment. The system later
> remembered what might have helped, but treated it as limited evidence instead
> of pretending it already knew the user.

The real phone clips prove the product works. The isolated Demo Lab then
compresses the longer recovery story. Architecture and code appear only after
the human value is clear.

## Recording assets

Prepare these before the final take:

1. A two-second shot of the real cup sketch or Chloe holding it beside the
   phone. Do not show unrelated personal surroundings.
2. Phone clip: real Goal Architect plan and approved support agreement.
3. Phone clip: confirmed real check-in decision and specific recognition.
4. Phone Developer clip: `gpt-5.6-sol`, Memory Curator, one retrieved safe
   memory ID, and no raw photo/reply.
5. Browser clip: cloud `/demo`, one-click Day 30, then Journey and Developer.
6. Browser clip: README architecture, Codex build log, dated decisions, tests,
   and Git commit history.
7. Final title card with project name, public Demo Lab, GitHub, and MIT.

Use slow cursor movement, large text, and 1080p or higher. Record phone and
desktop separately, then edit; do not depend on a live API call during the
final take.

## Timed English narration and screen actions

### 0:00–0:17 — Human hook

**Screen:** Real cup sketch beside the installed phone; cut to the confirmed
completion card.

**Narration:**

> I bought drawing supplies and kept planning to practice—but I had not drawn
> a single picture. Time Sovereignty gave me one protected twenty-minute
> commitment. I reported back with a real photo, and I actually drew this cup.

### 0:17–0:39 — The product promise and consent

**Screen:** Three onboarding answers → real Goal Architect plan → support
agreement controls.

**Narration:**

> This is not another planner. It is an AI Chief of Staff for the moment real
> life breaks the plan. I define the goal, deadline, and why it matters.
> GPT-5.6 proposes a goal-led cadence, but I approve the plan, quiet hours,
> tone, progress formats, pause rules, and when firmer follow-up is allowed.

### 0:39–1:12 — Real phone loop

**Screen:** Scheduled check-in → photo/text/voice form → structured decision →
confirm completion → Developer trace.

**Narration:**

> A real Google-signed Cloud Task brings the check-in back to the PWA. I can
> answer with text, voice, or a temporary photo. GPT-5.6 decides whether I am
> on track, partly done, blocked, changing direction, or complete. Recovery
> joins only when needed. I confirm the next commitment, and the app saves the
> decision, safe trace, memory proposal, and follow-up. The API key never
> reaches the phone, and the photo is not stored.

### 1:12–1:41 — Memory that earns trust

**Screen:** First/second memory acceptance summary; Developer shows one
retrieved ID and Memory Curator; optionally show confidence 0.35 → 0.47.

**Narration:**

> The first success becomes an immutable episode and only a tentative strategy:
> a short deadline and continuous lines may have helped me start. On a later
> real check-in, GPT-5.6 retrieved that card, explicitly treated it as limited
> evidence, changed its response, and updated effectiveness after the second
> outcome. One success never becomes a permanent claim about who I am.

### 1:41–2:12 — Thirty days in thirty seconds

**Screen:** Public `/demo` honest-boundary card → Run full 30-day story →
Journey Recovery/Calibration → Developer mock traces.

**Narration:**

> For judges, this clearly labeled Demo Lab compresses thirty days of an
> illustration habit into seconds. One delay is accepted. Repeated delay opens
> Commitment Recovery. The action gets smaller, progress receives specific
> recognition, memory carries forward, and the current goal can be recalibrated
> without discarding the deeper North Star. This lane is scripted, browser-only,
> and makes zero API calls.

### 2:12–2:42 — Codex and GPT-5.6 engineering proof

**Screen:** Architecture → safe traces → `125 passed` → build log/decisions →
Git history.

**Narration:**

> Codex was my primary engineering environment. From one PRD, it built the
> Next.js PWA, two state machines, four structured Agent contracts, Cloud Run,
> Cloud Tasks, Firestore, OIDC, Secret Manager, Android acceptance, tests, and

> serialized timestamps and stale PWA code to a Realtime cutoff and a cloud
> hydration mismatch. GPT-5.6 remains the real structured decision brain, with
> strict Zod outputs, safe traces, idempotency, and zero SDK retries.

### 2:42–2:56 — Vision and close

**Screen:** Today / completed cup → title card with Demo Lab and GitHub.

**Narration:**

> Today it can find me on my phone. Tomorrow it can meet me through a watch or
> home speaker—but only with my consent. Time Sovereignty does not demand
> perfect consistency. It helps one meaningful goal survive real life.
