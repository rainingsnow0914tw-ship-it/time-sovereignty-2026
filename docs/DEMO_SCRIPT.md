# Time Sovereignty — Under-3-Minute Demo Script

Target runtime: **2 minutes 50 seconds**. Hard stop: **2 minutes 58 seconds**.
The final upload must be a public YouTube video with audible narration that
explicitly explains both Codex and GPT-5.6.

## Before recording

1. Open the deployed app at
   `https://time-sovereignty-29309448808.asia-east1.run.app` in a 390×844 mobile
   viewport.
2. Confirm `/api/health` shows HTTP 200, provider `live`, model `gpt-5.6`, and
   the current revision.
3. Reset the browser journey once, then prefill or rehearse the short goal text.
4. Allow browser audio. Keep the text fallback visible; do not depend on a

6. Record at 1080p or higher. Keep the cursor slow and readable.

## Timed narration and screen actions

### 0:00–0:15 — The problem

**Screen:** Product opening and the first goal question.

**Say:**

> Most productivity tools make a plan once. Time Sovereignty is an AI Chief of
> Staff that protects the goal when real life interrupts, without shame or
> forcing the user to rebuild all the context.

### 0:15–0:40 — Consent before intervention

**Screen:** Enter a short goal, show the proposed plan, then move through the
support agreement.

**Say:**

> The user defines the goal, deadline, and why it matters. Goal Architect
> proposes a plan, but the user must approve both the plan and the support
> agreement: rhythm, quiet hours, tone, progress formats, pause conditions, and
> stronger follow-up consent.

### 0:40–1:12 — Recovery instead of blame

**Screen:** Today, Incoming Check-in, one delay, then a repeated delay and a
smaller recommitment.

**Say:**

> Today keeps the north star, protected minimum, next action, and exact resume
> point visible. One delay is accepted. A repeated delay activates Commitment
> Recovery, which asks whether timing, task size, method, or direction changed.
> The result is a smaller commitment and a preserved place to restart.

### 1:12–1:35 — Evidence and memory

**Screen:** Share text or photo progress, show specific feedback, then rate the
intervention.

**Say:**

> Progress can be text, photo, or voice. The system gives specific feedback,
> stores a structured progress memory, and asks whether the intervention
> actually helped instead of assuming success.

### 1:35–1:58 — Thirty days in seconds

**Screen:** Run the accelerated simulation and open Journey.

**Say:**

> This clearly labeled simulation compresses thirty days into seconds. Journey
> shows planning, interruption, adaptation, evidence, calibration, memory, and
> continuation as one longitudinal story.

### 1:58–2:28 — Real GPT-5.6 cloud proof

**Screen:** Developer, runtime status, safe traces, then the health and evidence
record.

**Say:**

> The browser journey is local-first, while this protected cloud path is real.
> A Google-signed Cloud Task called four production Agents through the OpenAI
> Responses API with GPT-5.6. Strict Zod schemas validated every output, and
> Firestore stored only safe traces. The run used 3,286 tokens. A duplicate task
> with the same request ID produced no second model call or cost.

### 2:28–2:48 — How Codex built it

**Screen:** Repository build log, decisions, tests, and evidence folders.

**Say:**

> Codex was the primary engineering environment. It turned the PRD into the
> product, tests, cloud infrastructure, deployment, debugging, and this dated
> evidence chain. The repository shows where Codex accelerated the work, where
> decisions changed, and what was verified in the real runtime.

### 2:48–2:58 — Close

**Screen:** Return to Today and the live URL.

**Say:**

> Time Sovereignty does not demand perfect consistency. It protects meaningful
> continuity. Your plan may change; your goal does not have to disappear.
